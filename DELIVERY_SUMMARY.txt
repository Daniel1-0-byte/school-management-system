💡 KEY DESIGN DECISIONS

1. No Photo Uploads for People
   → Only InitialsAvatar component used
   → Simplifies security and RLS
   → Cleaner data model

2. Shared Login Page for School Users
   → All roles (Admin, Teacher, Parent) use /login
   → Role detection via database
   → Simpler UI, consistent UX

3. Completely Separate Admin System
   → Platform admins use separate table
   → No crossover with school data
   → Enhanced security and isolation

4. Stateless Architecture
   → No background jobs or cron tasks
   → All state in Supabase
   → Perfect for Vercel free tier

5. Full TypeScript Coverage
   → No `any` types
   → Strong type safety
   → Fewer runtime errors
