Issue: TOTP Fallback Bypassing 2FA
Severity: CRITICAL
Status: FIXED

Before (INSECURE):
  if (!authenticator) {
    return code.length === 6 && /^\d{6}$/.test(code);  // ANY 6-digit code passes!
  }

After (SECURE):
  if (!otplib || !otplib.verify) {
    console.error('[v0] CRITICAL: TOTP verification failed - otplib not available');
    return false;  // REJECT all codes if unavailable
  }

Impact:
- Previously: If otplib failed to load, any 6-digit code would pass 2FA
- Now: If otplib unavailable, all 2FA attempts are rejected
- Result: No security bypass - complete protection
